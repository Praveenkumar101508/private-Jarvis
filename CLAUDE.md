# IRA (Intelligent Responsive Assistant) - Defacto Production Matrix

## 1. System Topology & Cross-Device Infrastructure Mapping
IRA is implemented as a 100% private, sovereign multi-agent ecosystem designed to run natively without container layers due to nested virtualization constraints on the compute host.

### Hardware Allocation & Routing Matrix:
- Thin Client Node (MacBook Air M1): Manages the primary FastAPI web gateway, the LiveKit WebSockets transport loop, local speech-to-text (faster-whisper), and local text-to-speech (kokoro ONNX). It hosts a fast, local `llama3.1:8b` via Ollama for immediate intent routing and voice conversation loops.
- Compute & Storage Node (Shadow PC - NVIDIA RTX A4500 20GB VRAM / 28GB RAM): Hosts the native PostgreSQL relational and vector instance, handles heavy agentic reasoning via `qwen2.5-coder:32b` under Ollama, and executes autonomous script validation runtimes.
- Encrypted Connection Tunnel: Communication channels between nodes must traverse a private mesh network tunnel (e.g., Tailscale static IPs). Local environment constants must bind directly to the secure mesh network layer.

## 4. Development Code-Gen, Verification, & GitHub Backup Loop

When building, modifying, or testing components in this repository, strictly adhere to this automated execution loop:

1. **Initialize**: Confirm all system configurations point to local loopbacks and designated secure Tailscale connection IPs.
2. **Validate Syntax**: Run static analysis check routines using local syntax linters and shellcheck prior to running the system execution loops.
3. **Automated Testing**: Run every generated microservice or routing node inside the local sandbox environment first.
4. **Autonomous Fixes**: If any runtime errors or connection exceptions are caught, intercept the trace logs, fix the source file dynamically, and re-test until the entire system passes 100% verification cleanly.
5. **Continuous GitHub Backup**: The absolute millisecond a module, sub-component, or phase passes all verification tests, immediately execute native git shell commands. Code changes must be committed dynamically with clear description flags (e.g., `git commit -m "feat: complete app/database.py initialization phase"`) and pushed to the remote branch instantly to secure development checkpoints.
